Выкладка сайта в интернет (форма RSVP шлёт в Telegram через Node server.js)
================================================================================

1) Залейте проект в GitHub (репозиторий публичный или приватный — как удобно).
   Файл .env в git не попадает — это нормально, секреты только на хостинге.

2) Render.com (бесплатный тариф, проще всего)
   - Зайдите https://render.com → Sign up (можно через GitHub).
   - New → Blueprint → подключите репозиторий с этим проектом.
   - Render подхватит render.yaml (или создайте Web Service вручную):
       Build:  npm install
       Start:  npm start
   - В разделе Environment добавьте переменные (как в .env дома):
       TELEGRAM_BOT_TOKEN = токен от @BotFather
       TELEGRAM_CHAT_ID   = ваш id (например от @userinfobot)
   - Deploy. Когда сборка зелёная — откройте выданный URL https://....onrender.com
   - На бесплатном плане сервис «засыпает» после простоя; первый заход может занять ~30–60 с.

3) Другие варианты с Docker
   - Любой хост с Docker: собрать образ из Dockerfile, при запуске передать те же env.
   - Railway / Fly.io: подключить репозиторий, команда старта npm start, переменные окружения те же.

4) После деплоя откройте сайт только по https-адресу хостинга (не file://).
   В index.html meta wedding-rsvp-api оставьте пустым — API на том же домене.

5) PHP-хостинг без Node
   - Загрузите файлы сайта + send-rsvp.php, на сервере создайте .env рядом с send-rsvp.php
     или задайте TELEGRAM_* в панели хостинга.
   - В script.js первым эндпоинтом идёт /api/rsvp — на чистом PHP-хостинге его нет; второй
     запрос пойдёт на /send-rsvp.php — убедитесь, что PHP включён и путь к скрипту верный.
